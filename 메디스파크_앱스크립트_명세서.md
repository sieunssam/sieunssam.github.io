# 메디스파크 랜딩페이지 Apps Script 명세서

## 1. 목적

- 랜딩페이지의 `상담하기`, `관심고객 등록`, `방문예약` 데이터를 Google Apps Script로 수집
- 수집된 데이터를 Google Sheets에 저장
- 필수 저장값: `이름`, `연락처`, `developer=박시은`

## 2. 구성 요소

### 프론트엔드

- 파일: `medispark_landing_form.html`
- 역할:
  - 사용자 입력 수집
  - 간단 유효성 검사
  - Apps Script Web App으로 POST 요청

### 백엔드

- 파일: `Code.gs`
- 역할:
  - JSON body 수신
  - 필수값 검증
  - 전화번호 정규화
  - 시트에 append
  - JSON 응답 반환

### 저장소

- 파일: `구글시트_컬럼샘플.csv`
- 역할:
  - 실제 시트 헤더 설계 기준

## 3. 전송 방식

- HTTP Method: `POST`
- Content-Type: `application/json`
- Endpoint: Apps Script Web App 배포 URL

예시:

```text
https://script.google.com/macros/s/REPLACE_WITH_DEPLOYMENT_ID/exec
```

## 4. 요청 payload

### 필수값

- `name`
- `phone`
- `developer`

### 권장값

- `consult_type`
- `interest_type`
- `visit_request`
- `visit_datetime`
- `page_url`
- `device`
- `utm_source`
- `utm_medium`
- `utm_campaign`
- `memo`

예시:

```json
{
  "name": "홍길동",
  "phone": "010-1234-5678",
  "developer": "박시은",
  "consult_type": "상담하기",
  "interest_type": "84A",
  "visit_request": "N",
  "visit_datetime": "",
  "page_url": "https://example.com/braincity",
  "device": "mobile",
  "utm_source": "naver",
  "utm_medium": "cpc",
  "utm_campaign": "braincity_launch",
  "memo": ""
}
```

## 5. 응답 형식

### 성공

```json
{
  "ok": true,
  "message": "saved",
  "row": 2
}
```

### 실패

```json
{
  "ok": false,
  "message": "name is required"
}
```

## 6. Google Sheets 컬럼 순서

1. `created_at`
2. `name`
3. `phone`
4. `developer`
5. `consult_type`
6. `interest_type`
7. `visit_request`
8. `visit_datetime`
9. `page_url`
10. `device`
11. `utm_source`
12. `utm_medium`
13. `utm_campaign`
14. `memo`

## 7. 유효성 검사 규칙

- `name`: 공백 제거 후 비어 있으면 실패
- `phone`: 숫자만 남긴 뒤 9자리 미만이면 실패
- `developer`: 반드시 `박시은`
- `visit_request`: `Y` 또는 `N`

## 8. 전화번호 정규화 규칙

- 저장 전 숫자만 남김
- 예:
  - `010-1234-5678` -> `01012345678`
  - `010 1234 5678` -> `01012345678`

## 9. 시트 준비 절차

1. 새 Google Sheets 생성
2. 첫 번째 시트 이름을 `leads`로 지정
3. 첫 행에 `구글시트_컬럼샘플.csv`의 헤더 입력
4. 시트 ID 복사
5. `Code.gs`의 `SPREADSHEET_ID` 수정

## 10. Apps Script 배포 절차

1. [script.new](https://script.new) 열기
2. `Code.gs` 내용 붙여넣기
3. 필요 시 `appsscript.json` 설정 반영
4. `SPREADSHEET_ID`, `SHEET_NAME`, `ALLOWED_DEVELOPER` 수정
5. `Deploy > New deployment`
6. Type: `Web app`
7. Execute as: `Me`
8. Who has access: `Anyone`
9. 배포 URL 복사
10. 프론트의 `APP_SCRIPT_URL` 값에 반영

## 11. 테스트 체크리스트

- 이름만 입력하고 제출 시 실패하는지 확인
- 연락처 형식이 섞여 있어도 정상 정규화되는지 확인
- developer 값이 `박시은`으로 저장되는지 확인
- Google Sheets 행이 실제로 append 되는지 확인
- 성공 시 프론트 메시지가 바뀌는지 확인
- 실패 시 에러 메시지가 보이는지 확인

## 12. 운영 메모

- CORS 이슈를 줄이려면 프론트는 `fetch` + `text/plain` 대신 현재 JSON 구조 유지
- 광고 유입 구분이 필요하면 UTM은 꼭 저장
- 개인정보 처리 고지 문구는 폼 하단에 별도 노출 권장

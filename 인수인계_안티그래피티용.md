## 4. 현재 페이지 상태 요약

현재 [index.html](C:/Codex/20260627_vibe/index.html)은 다음 상태입니다.

- 상업용 공개 랜딩 기본 구조 완료
- 노트북 구간에서 `PLAN SELECTOR` 영역 컴팩트 조정 완료
- 브라우저 코멘트 반영 완료
  - 히어로 타이틀 축소
  - 내부 메모성 섹션 제거/대체
- `상담하기 → 이름/연락처 입력 → 저장 → 상담 연결/전화 연결` 구조 구현
- `developer=박시은` hidden 값 포함
- UTM, page URL, device 저장 구조 포함
- 기본 스팸 방어 포함
  - honeypot
  - 너무 빠른 제출 차단

하지만 아래 항목은 아직 실제 운영값 입력 전 상태입니다.

- Apps Script 배포 URL placeholder 상태
- Google Sheets ID placeholder 상태
- 실제 실저장 테스트 미완료

## 5. 반드시 해야 하는 작업

이 프로젝트는 아래 4개를 해야 “운영 가능한 상태”가 됩니다.

### 1. Apps Script 실제 값 입력

수정 파일:

- [Code.gs](C:/Codex/20260627_vibe/Code.gs)
- [landing.config.js](C:/Codex/20260627_vibe/landing.config.js)

해야 할 일:

- `Code.gs`의 `SPREADSHEET_ID`를 실제 시트 ID로 변경
- `landing.config.js`의 `appScriptUrl`을 실제 Apps Script Web App URL로 변경

### 2. Google Sheets 준비

해야 할 일:

- 새 시트 생성
- 시트명 `leads`
- 헤더는 [구글시트_컬럼샘플.csv](C:/Codex/20260627_vibe/구글시트_컬럼샘플.csv) 기준으로 입력

### 3. Apps Script 배포

필수 조건:

- `Deploy > New deployment`
- Type: `Web app`
- Execute as: `Me`
- Access: 운영 환경에서 POST 가능한 범위

### 4. 실저장 테스트

테스트 기준 문서:

- [실저장_테스트_체크리스트.md](C:/Codex/20260627_vibe/실저장_테스트_체크리스트.md)

최소 확인:

- 히어로 폼 저장 성공
- 모달 폼 저장 성공
- 저장 후 전화걸기 정상 동작
- 시트에 `이름`, `연락처`, `developer=박시은` 저장 확인

## 8. 현재 반영된 폼 동작

### 히어로 상단 폼

파일 위치:

- [index.html](C:/Codex/20260627_vibe/index.html)

동작:

- 이름 입력
- 연락처 입력
- 관심 타입 선택 가능
- 바로 저장 요청

### 상담 모달

동작:

- `상담하기`, `관심고객 등록`, `방문예약`, `타입 상담하기` 버튼 클릭 시 오픈
- 이름/연락처 필수
- 관심 타입, 방문예약 여부, 방문 희망일시, 메모 선택 입력
- `상담 연결받기` 저장
- `저장 후 전화걸기` 저장 후 `tel:` 이동

### 스팸 방어

현재 반영:

- hidden honeypot `website`
- `form_started_at` 기반 너무 빠른 제출 차단

주의:

- 이것만으로 완전한 방어는 아님
- 운영 단계에서는 필요 시 reCAPTCHA 또는 서버 측 토큰 검증 추가 검토 가능

## 10. 공개 전 체크포인트

공개 전에 아래는 반드시 확인해야 합니다.

- 대표번호가 전 구간 `010-6689-2348`인지
- 고객 노출 문구에 내부 메모성 표현이 남아 있지 않은지
- `예정`, `계획`, `추진` 표현이 필요한 곳에 유지되는지
- 시트에 실접수 3건 이상 저장되는지
- `developer=박시은`이 실제 저장되는지
- 전화걸기 버튼이 저장 후 정상 동작하는지
- 노트북/모바일에서 `PLAN SELECTOR`가 과도하게 크지 않은지

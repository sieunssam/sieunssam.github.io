# 더미 제출 테스트 시나리오

## 시나리오 1. 상단 히어로 폼 정상 저장

입력값:

- 이름: `테스트1`
- 연락처: `010-1111-2222`
- 관심 타입: `84A`

기대결과:

- 성공 메시지 노출
- 시트에 한 줄 추가
- `consult_type=상담하기`
- `developer=박시은`

## 시나리오 2. 모달 상담하기 저장

입력값:

- 이름: `테스트2`
- 연락처: `010-2222-3333`
- 상담 유형: `상담하기`
- 관심 타입: `84B`
- 방문예약 여부: `N`

기대결과:

- 성공 메시지 노출
- 시트에 `84B` 저장

## 시나리오 3. 방문예약 저장

입력값:

- 이름: `테스트3`
- 연락처: `010-3333-4444`
- 상담 유형: `방문예약`
- 관심 타입: `101`
- 방문예약 여부: `Y`
- 방문 희망일시: `7/5 오후 2시`

기대결과:

- 시트에 `visit_request=Y`
- `visit_datetime=7/5 오후 2시`

## 시나리오 4. 저장 후 전화걸기

입력값:

- 이름: `테스트4`
- 연락처: `010-4444-5555`

기대결과:

- 시트 저장 후 전화 연결 시도
- 모바일에서 전화 앱 호출

## 시나리오 5. 유효성 실패

입력값:

- 이름: 빈 값
- 연락처: `010-5555-6666`

기대결과:

- `이름을 입력해주세요.` 메시지
- 시트 저장 없음

## 시나리오 6. 너무 빠른 제출 차단

방법:

- 모달 오픈 직후 1초 안에 즉시 제출 시도

기대결과:

- `잠시 후 다시 시도해주세요.` 또는 차단 메시지 노출

## 시나리오 7. UTM 저장

접속 URL 예시:

```text
file:///C:/Codex/20260627_vibe/index.html?utm_source=naver&utm_medium=cpc&utm_campaign=braincity_launch
```

입력값:

- 이름: `테스트UTM`
- 연락처: `010-7777-8888`

기대결과:

- 시트에 `utm_source=naver`
- `utm_medium=cpc`
- `utm_campaign=braincity_launch`

# 메디스파크 분양 홍보 랜딩 페이지 기획서

## 1. 기획 목적

- 목표: 브레인시티 메디스파크 로제비앙 모아엘가 분양 관심고객의 상담 문의와 DB 수집
- 1차 전환: `관심고객 등록`, `분양 일정/타입 안내 받기`, `상담 신청`
- 2차 전환: 대표번호 연결, 카카오/네이버 폼 연동, 모델하우스 방문 예약 유도
- 대표번호: `010-6689-2348`
- 기본 상담 플로우: `상담하기 클릭 → 이름/연락처 간단 입력 → 상담원 연결 또는 회신`
- 개발 담당자: `박시은`
- DB 저장 방식: `Google Apps Script → Google Sheets`

## 12. 개발 전달사항

- 랜딩페이지 개발 담당자는 `박시은`
- 모든 상담/관심고객 폼 제출 시 `developer=박시은` 값을 함께 전송
- 프론트엔드에서는 hidden input 또는 JS payload에 `developer: "박시은"` 고정 포함
- Google Apps Script Web App URL은 환경변수 또는 설정 상수로 분리 권장
- 시트 헤더명은 프론트 payload 키와 최대한 동일하게 맞춤
- 앱스크립트에서 `doPost(e)`로 JSON body를 받아 시트에 append하는 구조 권장
- 테스트 시 `이름`, `연락처`, `developer` 3개 값이 실제 시트에 남는지 우선 검증

## 13. 운영 시 주의사항

- 교통 및 개발 관련 표현은 반드시 `예정`, `계획`, `추진` 표현 유지
- 의료시설 및 개발 규모는 제공 자료 기준으로만 사용
- 과장 표현보다 `도보`, `세대수`, `거리`, `공급 구성` 등 확인 가능한 정보 중심으로 설계
- 랜딩 페이지 하단에 분양 자료 기준 시점과 유의 문구 병기 권장
- 참조 페이지와 PDF 사이에 사업개요 표기 차이가 있을 수 있으므로 공개 전 `동 수`, `입주예정`, `시행/시공`, `사업지 표기` 최종 팩트체크 필요
- 대표번호는 전 페이지 공통으로 `010-6689-2348` 단일화
- 개인정보 입력은 최소 항목 우선 원칙 유지
- `이름`, `연락처` 외 항목은 선택값 또는 2단계 입력으로 분리 권장

이 랜딩 페이지는 `브레인시티 일반 분양`이 아니라 `메디컬 프리미엄 + 도보 생활권 + 미래 교통가치`를 동시에 파는 구조로 가야 합니다.

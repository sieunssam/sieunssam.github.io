# 브레인시티 메디스파크 랜딩페이지 실저장 테스트 체크리스트

## 1. 배포 전 설정

- [ ] `Code.gs`의 `SPREADSHEET_ID`를 실제 Google Sheets ID로 변경
- [ ] Google Sheets에 `leads` 시트가 존재하는지 확인
- [ ] 시트 첫 행 헤더가 [구글시트_컬럼샘플.csv](C:/Codex/20260627_vibe/구글시트_컬럼샘플.csv)와 같은지 확인
- [ ] Apps Script를 `Web app`으로 배포
- [ ] 접근 권한이 실제 랜딩페이지에서 POST 가능한 상태인지 확인
- [ ] `landing.config.js` 파일을 생성하고 `appScriptUrl` 실배포 URL 입력
- [ ] `index.html`에서 `landing.config.js`를 먼저 로드하거나 인라인으로 `window.LANDING_CONFIG`를 선언

## 2. 기본 접수 테스트

- [ ] 히어로 상단 폼에서 `이름 + 연락처` 입력 후 접수 성공
- [ ] Google Sheets에 `name`, `phone`, `developer`가 정상 저장
- [ ] `developer` 값이 `박시은`으로 저장
- [ ] `consult_type` 값이 `상담하기`로 저장
- [ ] 전화번호가 숫자만 저장되는지 확인

## 3. 모달 접수 테스트

- [ ] `상담하기` 버튼 클릭 시 모달 오픈
- [ ] 이름 없이 제출 시 에러 메시지 노출
- [ ] 잘못된 연락처 입력 시 에러 메시지 노출
- [ ] 정상 입력 시 접수 성공 메시지 노출
- [ ] `visit_request`, `visit_datetime`, `interest_type` 저장 확인

## 4. 저장 후 전화걸기 테스트

- [ ] `저장 후 전화걸기` 클릭 시 먼저 저장 시도
- [ ] 저장 성공 후 `tel:010-6689-2348` 연결 시도
- [ ] 저장 실패 시 전화 연결 대신 에러 메시지 노출

## 5. 스팸 방어 테스트

- [ ] hidden honeypot 필드에 값이 들어가면 저장 거부되는지 확인
- [ ] 폼 오픈 직후 즉시 제출하면 `잠시 후 다시 시도해주세요` 또는 `submitted too quickly`가 발생하는지 확인
- [ ] 정상 입력 후 1.5초 이상 지난 제출은 허용되는지 확인

## 6. 유입정보 저장 테스트

- [ ] URL에 `?utm_source=naver&utm_medium=cpc&utm_campaign=braincity_launch` 붙여 접속
- [ ] 저장 후 시트에 `utm_source`, `utm_medium`, `utm_campaign` 값이 남는지 확인
- [ ] `page_url`, `device` 값이 함께 남는지 확인

## 7. 브라우저/디바이스 테스트

- [ ] 데스크톱 크롬에서 상단 폼 접수 확인
- [ ] 모바일 브라우저에서 하단 고정 CTA 동작 확인
- [ ] 노트북 해상도에서 `PLAN SELECTOR` 영역이 한눈에 들어오는지 확인
- [ ] 모바일에서 모달 입력창이 잘리지 않는지 확인

## 8. 공개 전 최종 확인

- [ ] 대표번호가 전 페이지에서 `010-6689-2348`로 통일되었는지 확인
- [ ] 고객용 문구에 내부 메모성 표현이 남아 있지 않은지 확인
- [ ] 교통/개발 관련 `예정`, `계획`, `추진` 표현 유지 여부 확인
- [ ] footer와 CTA 문구가 고객 관점으로 정리되어 있는지 확인
- [ ] 실접수 3건 이상 시트 적재 후 운영팀이 확인 완료
